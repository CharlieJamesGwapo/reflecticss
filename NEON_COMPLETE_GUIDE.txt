  STEP-BY-STEP SETUP
================================================================================

STEP 1: CREATE NEON ACCOUNT
  1. Visit https://neon.tech
  2. Click "Sign Up"
  3. Choose method (Email, GitHub, or Google)
  4. Complete sign-up
  5. Verify email
  6. Log in

STEP 2: CREATE PROJECT
  1. Click "New Project"
  2. Enter:
     - Project Name: css-review
     - Database Name: css_review (with underscore!)
     - Region: Choose closest to you
     - PostgreSQL: 15 (or latest)
  3. Click "Create Project"
  4. Wait for creation (< 1 minute)

STEP 3: GET CONNECTION STRING
  1. Click your project
  2. Click "Connection string" tab
  3. Copy the full connection string
  4. Save it somewhere safe

STEP 4: UPDATE BACKEND .env
  1. Open: backend/.env
  2. Find: DATABASE_URL=postgresql://localhost:5432/css_review
  3. Replace with: DATABASE_URL=postgresql://user:password@host.neon.tech/css_review?sslmode=require
  4. Save file

STEP 5: LOAD SCHEMA
  1. Install psql (if needed)
  2. Run: psql "your_connection_string" -f backend/database/schema.sql
  3. Wait for completion
  4. No errors = Success ‚úÖ

STEP 6: LOAD DATA
  1. Run: psql "your_connection_string" -f backend/database/seed.sql
  2. Wait for completion
  3. No errors = Success ‚úÖ

STEP 7: VERIFY
  1. Connect: psql "your_connection_string"
  2. Check: \dt (list tables)
  3. Count: SELECT COUNT(*) FROM lessons;
  4. Should show: 6
  5. Exit: \q

STEP 8: TEST
  1. Start backend: npm run dev
  2. Check: http://localhost:5000/health
  3. Should show: {"status":"OK"}
  4. Success! ‚úÖ

================================================================================
  TROUBLESHOOTING
================================================================================

PROBLEM: Connection Refused
  Error: Error: connect ECONNREFUSED
  Solution:
    1. Check internet connection
    2. Verify connection string is correct
    3. Copy fresh connection string from Neon
    4. Check Neon project is active

PROBLEM: SSL Error
  Error: SSL connection error
  Solution:
    Ensure connection string includes: ?sslmode=require
    Example: postgresql://...@host.neon.tech/db?sslmode=require

PROBLEM: Authentication Failed
  Error: password authentication failed
  Solution:
    1. Copy connection string again from Neon
    2. Verify username and password
    3. Check for special characters
    4. Escape special characters if needed

PROBLEM: Database Not Found
  Error: database "css_review" does not exist
  Solution:
    1. Create database in Neon console
    2. Verify database name is "css_review"
    3. Run schema.sql again

PROBLEM: Tables Not Found
  Error: relation "lessons" does not exist
  Solution:
    1. Run schema.sql again
    2. Verify no errors during execution
    3. List tables with \dt
    4. Check database name matches

PROBLEM: Backend Won't Start
  Error: Error: connect ECONNREFUSED
  Solution:
    1. Verify DATABASE_URL in .env is correct
    2. Check Neon connection string
    3. Verify internet connection
    4. Check Neon project is active
