================================================================================
  QUICK FIX - API ERROR ✅
================================================================================

THE PROBLEM:
  "Something went wrong!" error in Account Settings
  API URL not configured

THE SOLUTION:
  Create .env file in frontend folder

================================================================================
  STEP 1: CREATE .env FILE (1 minute)
================================================================================

Option A: Using Terminal (Recommended)

  cd frontend
  echo "REACT_APP_API_URL=http://localhost:5000" > .env.local

Option B: Manual

  1. Open frontend folder
  2. Create new file named ".env.local"
  3. Add this line: REACT_APP_API_URL=http://localhost:5000
  4. Save file

Option C: Using .env instead of .env.local

  cd frontend
  echo "REACT_APP_API_URL=http://localhost:5000" > .env

================================================================================
  STEP 2: RESTART FRONTEND (2 minutes)
================================================================================

Close the frontend terminal and run:

  cd frontend
  npm start

Wait for "Compiled successfully" message.

================================================================================
  STEP 3: TEST (2 minutes)
================================================================================

1. Go to http://localhost:3000
2. Login
3. Click Account Settings
4. Update name or email
5. Click "Save Changes"
6. Should see success message!

================================================================================
  IMPORTANT NOTES
================================================================================

1. .env.local is preferred over .env
   - .env.local is not tracked by git
   - .env might be tracked by git

2. You MUST restart frontend after creating .env
   - Close npm start
   - Run npm start again

3. Backend must be running
   - npm run dev in backend folder
   - Should show "Server running on port 5000"

4. Check browser console for errors
   - Press F12
   - Go to Console tab
   - Look for red error messages
